# Telegram Alert

Self-service Telegram-бот для стримеров: подключает группу или канал, выдаёт персональную OBS-ссылку, меняет оформление и запускает тестовое уведомление без участия владельца сервиса.

## Сценарий стримера

1. Открыть бота и нажать `/start`.
2. Нажать большую кнопку выбора группы либо отдельную кнопку канала.
3. Выбрать Telegram-группу или канал, где пользователь может добавить бота администратором.
4. Скопировать выданную ссылку в OBS как Browser Source размером `520 × 120`.
5. Нажать **Проверить уведомление**.

Больше ничего передавать владельцу сервиса не нужно. Команда `/panel` в любой момент открывает подключённые чаты, OBS-ссылки и выбор оформления.

## Возможности

- несколько независимых стримеров, групп и каналов на одном деплое;
- отдельная случайная OBS-ссылка для каждого чата;
- стили **Графит**, **Светлый** и **Только текст**;
- мгновенное применение стиля без перезагрузки OBS;
- тест подписчика прямо из Telegram;
- отключение группы или канала и выход бота через подтверждение;
- проверка владельца при любом управляющем callback;
- автоматическая регистрация webhook и команд после первого деплоя Render.

Telegram официально поддерживает выбор группы или канала через `KeyboardButtonRequestChat`. Бот запрашивает минимальные права администратора, необходимые для получения `chat_member`.

## Первый деплой на Render

1. Создайте бота через `@BotFather`.
2. В Render выберите **New → Blueprint**.
3. Подключите репозиторий `Clownnotinbox/telegram-alert` и подтвердите `render.yaml`.
4. В единственное обязательное секретное поле вставьте `BOT_TOKEN`.
5. Нажмите **Deploy Blueprint**.

Blueprint сам создаёт PostgreSQL, `TELEGRAM_WEBHOOK_SECRET`, `ADMIN_KEY` и выполняет `npm run telegram:setup` после первого успешного деплоя. `RENDER_EXTERNAL_URL` предоставляется Render автоматически.

После статуса **Live** достаточно отправить стримеру ссылку `https://t.me/USERNAME_БОТА`. Доступ к Render, ключам и панели сайта ему не нужен.

## Локальная разработка

Нужен Node.js 22.13 или новее.

```bash
cp .env.example .env.local
npm install
npm run dev
```

Для локального интерфейса Telegram-токен не обязателен. Для проверки настоящего бота укажите `BOT_TOKEN` только в `.env.local`; этот файл исключён из Git.

## Переменные окружения

| Переменная | Назначение |
| --- | --- |
| `BOT_TOKEN` | единственный вводимый вручную секрет от `@BotFather` |
| `TELEGRAM_WEBHOOK_SECRET` | генерируется Blueprint для проверки webhook |
| `ADMIN_KEY` | резервный ключ ручного восстановления webhook |
| `PUBLIC_URL` | необязательный URL вне Render |
| `DATABASE_URL` | автоматически подключённый PostgreSQL |

## Хранилище

Render использует PostgreSQL. Локальная разработка использует D1, а тесты могут использовать временное хранилище в памяти. Установки и события разделены по внутреннему `installation_id`; публичный оверлей открывается только по случайному `overlayKey`.
